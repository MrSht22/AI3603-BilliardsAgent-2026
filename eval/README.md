# Eval 目录说明

本目录包含项目最终评估相关的核心文件和开发过程中的尝试记录。

---

## 核心文件

### 1. `evaluate.py` - 主评估脚本
项目的主要评估入口，用于让两个 Agent 进行多局对战并统计胜负。

### 2. `new_agent.py` - 最终 Agent 实现
我们项目最终整合到主函数的 Agent 实现。

**决策流程简介：**
NewAgent 采用物理计算与噪声感知相结合的策略。首先通过几何计算生成候选击球方案（遍历目标球-袋口组合，计算瞄准点、击球角度和力度），然后为每个基础方案生成多个变种（调整力度和角度）以应对环境噪声。接着使用蒙特卡洛模拟评估每个变种在带噪声条件下的期望得分，综合物理得分和模拟得分选择最优动作。当无好球可打时，执行安全球策略。

## 运行配置

### 超参数设置

在 `evaluate.py` 中，NewAgent 的超参数通过构造函数传入：

```python
agent_b = NewAgent(
    variant_config='BASELINE',    # 变种生成策略
    scoring_config='BASELINE'      # 评分函数配置
)
```

**可选配置：**

- **variant_config**: 控制变种生成策略
  - `'BASELINE'`: 基础配置（5个变种，无旋转）
  - `'AGGRESSIVE'`: 激进配置（7个变种，包含旋转，力度/角度偏移更大）
  - `'CONSERVATIVE'`: 保守配置（7个变种，包含旋转，力度/角度偏移更小）

- **scoring_config**: 控制评分函数权重
  - `'BASELINE'`: 平衡的奖惩配置
  - `'HIGH_REWARD_LOW_PENALTY'`: 高奖励低惩罚（鼓励进攻）
  - `'LOW_REWARD_HIGH_PENALTY'`: 低奖励高惩罚（鼓励保守）

### 运行命令

```bash
conda activate poolenv
cd AI3603-Billiards
python evaluate.py
```

### 环境配置

无需额外调整，使用项目默认环境配置即可。

## 开发历程

### `new_agent_previous_trial.py` - 尝试过程记录

本文件记录了 Agent 的迭代开发过程：

#### **NewAgent1** - 基础物理计算
- 实现了基于几何计算的精确击球逻辑
- 包含路径遮挡检测、切球角度计算、力度估算等基础功能
- 为后续改进奠定了物理计算框架

#### **NewAgent2** - 物理计算 + 贝叶斯微调
- 在 NewAgent1 基础上增加了贝叶斯优化
- 尝试通过仿真微调击球参数
- 探索了参数空间优化的可能性

#### **NewAgent3** - 带安全球策略的完整实现
- 实现了噪声感知的多变种评估机制
- 添加了复杂的安全球（safety ball）策略
- **最终失败原因**：安全球策略过于复杂，导致决策时间过长且效果不稳定

#### **NewAgent** (最终版本)
- 保留了 NewAgent3 的核心物理计算和噪声感知机制
- 简化了安全球策略，提高了决策效率
- 在性能和稳定性之间取得了良好平衡

## 评估说明

- 默认对战局数：120 局（可在 `evaluate.py` 中修改 `n_games` 参数）
- 自动轮换先后手和球型分配，确保公平性
- 统计指标：胜局数、平局数、总得分

## 使用建议

1. **对比测试**：修改 `evaluate.py` 中的配置参数，观察不同策略的表现差异
2. **调整对手**：在 `evaluate.py` 中切换 `BasicAgent` 或 `BasicAgentPro` 作为对手
3. **参数调优**：根据实际对战结果，在 `new_agent.py` 的配置类中调整超参数
